常见问题

什么是 Unix 时间戳？▼
Unix 时间戳是从 1970年1月1日 00:00:00 UTC 到指定时间的总秒数，也称为 Epoch 时间。它是计算机系统中最常用的时间表示方式，几乎所有编程语言都支持时间戳的读写。
秒、毫秒、微秒、纳秒有什么区别？▼
1秒 = 1,000毫秒 = 1,000,000微秒 = 1,000,000,000纳秒。不同系统和语言使用不同精度：JavaScript 使用毫秒、Java 使用毫秒、Python 默认秒（datetime支持微秒）、Go 支持纳秒。转换时注意乘除对应倍数即可。
为什么我的时间戳转换结果不对？▼
最常见的原因是精度不匹配。例如 JavaScript 的 Date.now() 返回毫秒级时间戳（13位数字），而大多数 API 接口使用秒级时间戳（10位数字）。如果输入13位数字但选了"秒"精度，结果会变成几万年以后。请确认精度选项与输入值匹配。
时间戳会溢出吗？▼
32位有符号整数存储的秒级时间戳会在 2038年1月19日 03:14:07 UTC 溢出（即"2038年问题"）。现代系统已普遍采用64位整数，可表示约2920亿年的时间范围，实际上不会溢出。
不同时区的时间戳一样吗？▼
是的。Unix 时间戳始终基于 UTC，与时区无关。同一个时刻，全球任何地方的时间戳值都相同。时区只影响"人类可读的日期时间"的表示方式。
